# AGENTS.md

このリポジトリで Codex が作業するときの共通ルールです。

## 基本方針

- 変更前に目的と影響範囲を確認し、既存の設計・命名・スタイルに合わせてください。
- 必要最小限の変更にとどめ、関係のないファイルは編集しないでください。
- 可能な限り、実装後にテストや静的チェックを実行してください。
- 専門用語を使う場合は、非エンジニアにも伝わるように短い補足を添えてください。

## セキュリティ

- API キー、トークン、パスワード、個人情報などの秘密情報をコードやコミットに含めないでください。
- 秘密情報が必要な場合は、環境変数や安全なシークレット管理の利用を案内してください。
- 外部サービス、ネットワーク、ファイル削除、破壊的操作を行う前に、必要性と影響を確認してください。

## Git / PR

- コミット前に `git status` で差分を確認してください。
- PR には、変更内容と実行した確認コマンドを簡潔にまとめてください。
